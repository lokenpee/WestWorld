# WestWorld

WestWorld 是一个 SillyTavern 扩展，提供 TXT 转世界书、章节节拍、导演注入与章节体验能力，目标是让小说互动体验更连贯。

## 1. 环境要求

- SillyTavern `>= 1.14.0`
- Node.js `>= 20`（仅在你要本地运行/开发 SillyTavern 时需要）
- 可用的模型 API（主 API + 导演 API，可同源也可分开）

## 2. 安装方式

### 2.1 在 SillyTavern 中安装（推荐）

1. 打开 SillyTavern。
2. 进入扩展管理界面。
3. 通过仓库地址安装第三方扩展（WestWorld 仓库地址）。
4. 安装后刷新页面。

### 2.2 本地目录安装（开发/调试）

1. 将本项目放到 SillyTavern 扩展目录：
   - `SillyTavern/public/scripts/extensions/third-party/WestWorld`
2. 刷新 SillyTavern 页面。
3. 在顶部扩展区确认出现 WestWorld 图标。

## 3. 首次配置顺序（重要）

请按顺序配置，避免“能打开但无法处理”的情况：

1. 打开 WestWorld 面板。
2. 先配置主 API：
   - Provider
   - API Key
   - Endpoint
   - Model
   - Max Tokens（建议先用默认值）
3. 再配置导演 API（可与主 API 相同，也可独立）：
   - 建议先启用 `导演开关`
   - 建议保留 `导演失败回退到主 API`
4. 配置章节识别：
   - 推荐使用默认章节正则
   - 小说章节格式特殊时再微调
5. 保存设置。

## 4. 运行流程

### 4.1 TXT 转世界书

1. 选择 TXT 文件。
2. 点击开始处理。
3. 等待队列完成。
4. 在结果区查看并导出世界书/角色卡。

### 4.2 章节体验模式

1. 进入“当前章节概览”。
2. 选择章节并发送开场白（自动或手动）。
3. 正常聊天推进剧情。
4. 导演注入会在生成前执行，确保节拍边界与承接。

### 4.3 导入/导出工程包

- 导出：用于保存当前处理进度、设置、章节状态。
- 导入：可恢复历史工程包继续处理。

## 5. 升级与兼容说明

本版本已从 `StoryWeaver` 升级为 `WestWorld`，并保留了兼容层：

- 旧 localStorage 配置键会自动识别。
- 旧全局对象名仍可访问（兼容旧脚本）。
- 旧导演注入标记会被识别并清理。
- 旧任务状态类型可继续导入。
- IndexedDB 会优先使用新库名，若检测到旧库则自动回退读取。

## 6. 常见问题

### 6.1 面板能打开但点“开始”无反应

- 检查主 API 是否完整配置。
- 检查模型是否可用。
- 检查 Max Tokens 是否在合法范围。

### 6.2 导演没有介入

- 确认导演开关已启用。
- 确认当前在有效章节与节拍上下文中。
- 检查是否处于静默/后台生成场景。

### 6.3 升级后像是“丢设置”

- 先刷新页面。
- 再打开设置确认是否从旧键迁移成功。
- 若仍异常，可导入旧工程包恢复。

## 7. 开发提示

- 入口文件：`index.js`
- 主模块：`txtToWorldbook/main.js`
- 导演服务：`txtToWorldbook/services/directorService.js`
- 设置持久化：`txtToWorldbook/services/settingsPersistenceService.js`
