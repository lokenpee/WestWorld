## 3. 布局与层级结构改动

相比原始 WestWorld 插件的扁平折叠列表，预览页对**提示词配置区**做了视觉层级重构：

### 原始问题
原始 UI 将以下六个元素用完全相同的折叠卡片样式平铺堆叠：
1. 默认世界书条目
2. 世界书词条
3. 剧情大纲
4. 文风配置
5. 消息链配置
6. 自定义提取分类

导致用户无法直观区分哪些是核心指令、哪些是可选附加、哪些根本不是提示词。

### 重构方案
- **默认世界书条目**: 独立为一个可折叠的 `ttw-prompt-section`，与提示词配置区平级，并用 `📚` 图标 + "可选" badge 弱化其存在感。
- **提示词配置**: 用 `ttw-section-header` 做成一个不可折叠的**大标题栏**，内部包含四个真正的提示词子项：
  - **世界书词条**: 带有蓝色高亮 "必需" badge，默认展开，强调其主导地位。
  - **剧情大纲 / 文风配置**: 带有复选框 + "可选" badge，默认折叠。
  - **消息链配置**: 标题文字颜色弱化（`--text-secondary`），明确其为"格式配置"而非"内容指令"。
- **自定义提取分类**: 标题颜色同样弱化为 `--text-secondary`，暗示它本质上是**数据/schema 定义**，不属于提示词本身。

## 4. 交互组件样式

### 标签页导航（`.ttw-view-tab`）
- 位于模态框头部下方，采用**圆角胶囊式顶栏标签**
- 未选中: 透明背景，`--text-tertiary` 文字
- 悬停: `--bg-hover` 背景
- 选中: `--bg-panel` 背景 + `--border` 边框 + `font-weight: 600`

### 折叠面板（`.ttw-prompt-section`）
- 头部: `padding: 10px 12px`，背景 `--bg-card`
- 悬停: 背景变为 `--bg-hover`
- 箭头图标: 默认 `▶`，展开时通过 `transform: rotate(90deg)` 旋转

### 上传区域（`.ttw-upload-area`）
- 2px 虚线边框 (`--border-highlight`)
- 悬停/拖拽进入: 边框变 `--accent`，背景变 `--bg-hover`

### 进度条（`.ttw-progress-bar`）
- 高度 10px，圆角 5px
- 填充色为 `--accent`，带 0.4s 宽度过渡动画

### 滚动条
- 宽度 6px
- track: `--bg-page`
- thumb: `--border-highlight`，圆角 3px

## 5. 新增的视觉辅助

- **Badge 系统**: 用三种样式区分重要性
  - 灰色 `可选`：`rgba(255,255,255,0.08)` 背景
  - 蓝色 `必需`：`rgba(37,99,235,0.15)` 背景 + `#93bbff` 文字
- **占位符提示块（`.ttw-placeholder-hint`）**: 浅高亮边框卡片，用于强调 `{DYNAMIC_JSON_TEMPLATE}` 的重要性
- **状态色块**: 章节队列中的条目通过左侧 3px 竖条颜色一眼区分"等待"/"完成"状态

## 6. 最新调整（2026-04-14 近期迭代）

### 6.1 标签重命名
- **默认世界书条目** → **向世界书中添加默认条目**
  - 作用更直白：说明这里是用来“预置默认条目”的，而非配置提示词。
- **世界书词条** → **txt转世界书主要提示词**
  - 避免与“世界书条目”混淆，明确这是“主 System Prompt”。

### 6.2 提示词配置区改为真正的父子容器
之前虽然把「📝 提示词配置」做成了标题栏，但子项与它仍是视觉并列关系（无缩进）。

**修改做法**:
- 将 `ttw-section-content` 的左内边距设为 `padding: 12px 12px 6px 22px`
- 所有子面板（主要提示词、提取分类、剧情大纲、文风配置、消息链配置）全部缩进在这个内容区内
- 从视觉上明确：它们都是「提示词配置」的子项，不再产生平级歧义

### 6.3 提取分类平铺化 + 删除「自定义提取分类」
原始 UI 把「角色 / 地点 / 组织 / 道具 / 章节剧情」塞在「自定义提取分类」的折叠面板里，导致：
1. 高频分类需要多点一次才能勾选
2. 和真正的提示词配置混在一起，层级混乱

**修改做法**:
- **删除「自定义提取分类」整个折叠区块**及其内部内容
- 将五大分类提取出来，在「txt转世界书主要提示词」下方放置一个**平铺的复选框卡片**：
  - 背景: `--bg-card`
  - 边框: `1px solid var(--border)`
  - 圆角: `8px`
  - 内部使用 `flex-wrap: wrap` 横向排列复选框
  - 顺序固定为：**👤 角色 → 📍 地点 → 🏛️ 组织 → ⚔️ 道具 → 📜 章节剧情**
- 这些分类**不可折叠**，直接可见、直接勾选，降低操作成本
- 保留内置分类的默认状态：角色/地点/组织 默认勾选；道具/章节剧情 默认不勾选

### 6.4 最终层级结构（当前）
```
📚 TXT转世界书
├── 📚 向世界书中添加默认条目（折叠面板，可选）
└── 📝 提示词配置（父容器）
    ├── 📚 txt转世界书主要提示词（折叠面板，必需，默认展开）
    ├── 🏷️ 提取分类（平铺复选框卡片，不可折叠）
    ├── 📖 剧情大纲（折叠面板，可选，带复选框）
    ├── 🎨 文风配置（折叠面板，可选，带复选框）
    └── 💬 消息链配置（折叠面板，可选）
```
