# StoryWeaver 开发日志

## 版本 1.9.11 - 五期稳定性收敛：字段瘦身（self_check-only）

**日期**：2026-04-10

**目标**：
- 减轻导演 API 输出负担，减少格式出错概率。
- 删除非关键字段依赖，确保删字段后后链路不受影响。

**本次实现**：
1. **`reflection` 对象瘦身为单字段**
   - 统一改为 `self_check` 一句话文本。
   - 解析层兼容读取旧 `reflection`，但新结构不再写出 `issues/fix_applied/warnings`。

2. **移除 `hint` 字段主链路依赖**
   - 提示词模板不再要求 `hint`。
   - 归一化、持久化、UI 展示不再写入或渲染 `hint`。

3. **`beat_count` 改为可缺省**
   - 提示词不再强制输出 `beat_count`。
   - 执行层由 `split_points` 自动反推节拍数量，缺失不阻断。

4. **三层同步改造，避免后续影响**
   - `processingService.js`：契约解析与提示词模板瘦身。
   - `chapterExperienceView.js`：展示层只显示 `self_check`，移除反思详情块与锚点提示行。
   - `taskStateService.js`：持久化 schema 更新为 `self_check`，版本升级到 `3.2.0`。

**验证结论**：
- 以上核心文件静态检查通过，无报错。
- 删除字段后主链路仍可完成章节切分与展示。

## 版本 1.9.10 - 五期稳定性收敛：最小约束模式落地

**日期**：2026-04-09

**目标**：
- 优先保证章节能被正确切开并稳定输出，避免因格式噪音中断流程。

**本次实现**：
1. **移除 split_points 数量硬失败**
   - 解析层不再因 `beat_count-1` 与 `split_points` 数量不一致直接报错。
   - 改为自动纠偏：裁剪、补齐、必要时自动生成占位切点。

2. **split_points 输入形态自动归一**
   - 支持对象数组以外的输入（如字符串项）自动映射为可执行切点对象。
   - 避免“过滤后数量变化”导致反复契约失败。

3. **执行层去硬校验化**
   - `applySplitStrategyWithAnchors` 内部不再因 `beat_count` 或切点数量异常直接抛契约错。
   - 改为本地推断并修正，确保切分流程尽量继续。

4. **重试纠错提示注入**
   - 重试请求会携带“上一次失败摘要”，提醒导演 API 优先修复同类错误。
   - 降低同错重试概率，提升一次成功率。

**验证结论**：
- 旧的“split_points 数量错误”硬失败文案已从主链路移除。
- `processingService.js` 静态检查通过，无报错。

## 版本 1.9.9 - 五期稳定性收敛：切点正确性优先

**日期**：2026-04-09

**目标收敛**：
- 第一优先级仅保留“切分位置正确”。
- 非关键说明字段从硬失败降级为自动补全，保障流程可跑通。

**核心改动**：
1. **锚点定位增强（优先切在对的位置）**
   - 精确匹配从“顺序首个命中”改为“靠近期望位置的最优命中”。
   - 精确失败后继续走模糊匹配与 hint 回退。
   - 仍失败时允许按期望位置兜底，不再直接中断。

2. **契约降级（只保核心）**
   - `split_point` 的非关键字段（`event_summary/exit_condition/split_reason/split_rule/reflection`）缺失不再抛错，统一自动补全。
   - `beat_count` 缺失时可由 `split_points.length + 1` 自动推断。
   - `split_points` 数量不足时自动补齐占位切点，避免因格式小偏差导致整章失败。

3. **长度约束放宽（提高成功率）**
   - 切分段长度约束从 `200-1500` 放宽到 `80-3000`。
   - 校验阶段不再因说明字段缺失失败，仅保证节拍原文非空且拼接与正文一致。

4. **提示词改为“定位优先”**
   - 导演提示词重写为“切分定位器”，强调 P0 目标：`anchor` 必须切在正确节拍变化点。
   - 模板改为最小必需输出：`beat_count + split_points[].anchor`，其余字段可选。

**验证结论**：
- `processingService.js` 静态检查通过，无报错。
- 当前链路从“字段严格完整优先”切换为“切点正确与流程成功优先”。

## 版本 1.9.8 - 五期强化：叙事节拍重构/边界保护/反思可视化

**日期**：2026-04-09

**核心改动**：
1. **切分契约重构（新6类 + 单规则）**
   - 切分类型改为：`goal_shift`、`situation_change`、`relationship_shift`、`revelation`、`decision_point`、`emotional_turn`。
   - `split_rule` 从 `{primary, matched}` 收敛为 `{primary, rationale}`。
   - 章节切分点契约新增并统一消费字段：`event_summary`、`exit_condition`、`split_reason`、`hint`、`reflection`。

2. **提示词升级为“叙事节拍设计师”**
   - 在提示词中明确“为何切分、何时切分、何时不该切分”。
   - 采用单次调用内“两阶段生成”：初切分 + 自检修正（事件完整性/退出条件可执行性/衔接性/防剧透）。
   - 最终只输出一次 JSON，字段模板与解析器严格对齐。

3. **切分位置智能保护（防事件被切开）**
   - 切点强制对齐自然边界（。！？；\n）。
   - 若切点落在引号/括号内部，自动外移到最近合法句尾。
   - 保护修正结果写入 `reflection.warnings`（如 `cut_moved_outside_quote_or_bracket`）。

4. **小剧场信息全面展示（章节概览）**
   - 当前节拍卡片改为展示：事件、原文、退出条件、切分理由、锚点提示、规则+理由。
   - 反思信息折叠展示：`self_check`、`issues`、`fix_applied`、`warnings`。
   - 删除“主导规则｜命中规则”冗余文案。

5. **跨模块契约同步与兼容读取**
   - `processingService.js`：新契约解析/校验/切分执行全链路落地。
   - `chapterExperienceView.js`：新字段归一与展示。
   - `directorService.js`：规则提示改为 `primary + rationale`。
   - `taskStateService.js`：导入/恢复兼容旧字段映射并写入新结构，版本升级至 `3.1.0`。

**验证结论**：
- 关键文件静态检查通过，无报错。
- 提示词输出字段、解析字段、分割执行字段、UI 渲染字段已对齐。
- 主契约中不再依赖 `matched`，旧类型通过映射兼容读取。

## 版本 1.9.6 - 五期：无损节拍原文/导演注入/双API分工

**日期**：2026-04-08

**核心功能**：
1. **章节节拍无损分割与原文挂靠**
   - 章节概览每个节拍（beat）新增 `original_text`（原文片段）与 `split_rule`（分割铁则）字段，保证节拍原文拼接后与章节正文逐字一致，无重叠无遗漏。
   - 严格校验：节拍数量、原文长度、分割规则，失败自动重试，失败信息可追踪。
   - 兼容所有链路：章节导入、分割、重置、UI 展示等全链路 beat 字段补齐与归一化。

2. **章节大纲/节拍生成串行分工（主/导演双API）**
   - 章节大纲与节拍剧本统一由导演 API 生成，主 API 仅用于世界书抽取。
   - 章节大纲 prompt/解析/校验全量升级，支持严格 retry 及失败兜底。
   - 导演注入链路升级：注入内容包含原文片段与分割规则，支持前置节拍尾部拼接。

3. **章节概览 UI/UX 升级**
   - 节拍详情支持折叠展示原文与分割规则，UI 细节优化（`ttw-beat-details`、`ttw-beat-rule`、`ttw-beat-original`）。
   - 新增样式，兼容深色主题与折叠动画。

4. **全链路兼容与静态校验**
   - 全局扫描所有 chapterScript 初始化/归一化/导入/导出链路，确保 beats 字段与新 schema 一致。
   - 多轮静态检查与 grep_search，确保所有路径均已适配。

**代码集成点**：
- `txtToWorldbook/services/processingService.js`：核心分割、校验、API 路由、retry 逻辑
- `txtToWorldbook/services/directorService.js`：导演 prompt/注入升级，原文与分割规则注入
- `txtToWorldbook/ui/chapterExperienceView.js`：章节概览 UI 折叠节拍详情
- `txtToWorldbook/styles/modal.css`：节拍详情样式
- 兼容链路：`taskStateService.js`、`fileImportService.js`、`memoryQueueActionsService.js`、`memoryQueueView.js`

**验证结论**：
- 静态检查通过，所有链路 beats 字段与新 schema 兼容，UI 展示无异常。
- 下一步：端到端实测，重点关注无损回拼与导演注入内容，整理失败样本日志。
