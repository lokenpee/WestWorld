# StoryWeaver 开发日志

## 版本 1.6.0 - 章节体验MVP（二期基础）

### 修改日期
2026-04-06

### 版本说明
此版本对应 git tag `v1.6`，实现二期核心功能的基础闭环。

### 主要功能
1. **章节大纲自动生成** - 每章处理时自动生成大纲和剧本(goal/flow/keyNodes)
2. **故事大纲页面** - 查看整书章节列表及大纲生成状态
3. **当前章节概览** - 显示本章摘要/剧本/开场白
4. **开场白自动发送** - 进入章节时自动生成并发送到聊天框
5. **章节切换** - 支持"下一章"切换，自动发送新章开场白

### 关键实现文件
- `services/processingService.js` - 章节大纲生成与重试逻辑
- `ui/chapterExperienceView.js` - 故事大纲与章节概览UI
- `ui/settingsPanel.js` - 添加入口按钮和容器
- `core/state.js` - 新增experience状态管理

### 数据模型
每个memory(章节)新增字段：
- `chapterOutline` - 章节摘要
- `chapterScript` - 剧本(goal/flow/keyNodes)
- `chapterOutlineStatus` - 生成状态(pending/done/failed)
- `chapterOpeningPreview` - 开场白预览
- `chapterOpeningSent` - 是否已发送

### 后续优化
- UI/UX细节打磨
- 章节结束自动判定
- 更多错误恢复机制

### 修改文件清单

#### 1. manifest.json
**变更内容**：
- 添加 `"html": "drawer-component.html"` 字段
- 版本号从 `0.1.0` 更新为 `0.2.0`

**原因**：SillyTavern 的抽屉组件需要通过 `html` 字段加载模板文件

#### 2. index.js（重写）
**主要变更**：
- 挂载位置：从 `$('#extensions_settings2').append(html)` 改为挂载到 `#extensions-settings-button` 后面
- 使用 `renderExtensionTemplateAsync` API 加载抽屉组件模板
- 新增抽屉开关逻辑（点击按钮展开/收起面板）
- 新增点击外部区域自动关闭抽屉功能

**保持不变的功能**：
- `ensureSettings()` - 设置初始化
- `initTxtToWorldbookBridge()` - txtToWorldbook 模块初始化
- `getTxtToWorldbookApi()` - API 获取
- `openTxtToWorldbookPanel()` - 打开转换器面板
- `window.StoryWeaver` 全局对象暴露
- 设置持久化 (`extension_settings.storyweaver`)

#### 3. style.css（重写）
**设计变更**：
- 新增深色主题 CSS 变量（`--sw-bg-darker`, `--sw-bg-dark`, `--sw-primary-accent` 等）
- 抽屉组件样式（位置、动画、阴影效果）

- 卡片式布局（简介卡片、操作区卡片）
- 现代化按钮样式（渐变背景、悬停效果、阴影）

#### 4. drawer-component.html（新增）
**组件结构**：
- 根元素 `#storyweaver-wrapper` + `class="drawer"`
- 触发按钮 `.drawer-toggle` + `#storyweaver-icon`（使用书本图标）

## 版本 1.5.0 - 一期需求落地（去重/整理/token/角色分级）

### 修改日期
2026-04-06

### 版本说明
此版本对应 git tag `v1.5`，是一期需求的核心功能落地版本。

### 对应需求
1. 文件上传区域移动到面板最顶端（位于 API 设置之前）
2. 别名合并与重复条目处理重做（优先自动处理明显同名冲突）
3. 长章节处理时 token 控制优化（避免注入整本世界书）
4. 角色条目新增四级分类（主角/重要配角/普通配角/NPC）并贯通导出

### 关键实现

#### 1) UI 顺序调整
- 文件：`txtToWorldbook/ui/settingsPanel.js`
- 调整 `buildModalBodyHtml()` 的区块顺序：`文件上传` 提前到第一块。

#### 2) 别名合并重做
- 新增文件：`txtToWorldbook/services/nameNormalizationService.js`
   - 提供条目名归一化、显式同名判断、内容近重复检测、内容去重合并。
- 文件：`txtToWorldbook/services/mergeService.js`
   - 新增“明显同名冲突”自动合并能力（含 `_卷N` / 常见后缀归一化）。
   - 升级短名匹配和重复检测逻辑，降低漏检率。
   - 合并时不再简单拼接，改为去重合并内容并删除冲突条目。
- 文件：`txtToWorldbook/services/mergeWorkflowService.js`
   - 在 AI 两两判断前加入预处理：自动合并明显重复组。
   - 当预处理已消除重复时直接提示完成，减少无效 AI 调用。

#### 3) 整理条目重做（结构化去重）
- 文件：`txtToWorldbook/services/mergeWorkflowService.js`
   - `consolidateEntry` 增加结构化输出约束。
   - 新增本地 `dedupeStructuredContent`：对“字段: 值”内容进行字段级去重。
   - 整理完成提示修正为成功/失败分级提示。

#### 4) Token 控制优化
- 文件：`txtToWorldbook/services/processingService.js`
   - 串行处理路径移除“当前世界书全量 JSON 注入”。
   - 改为注入“相关世界书摘录（精简）”，显著降低 prompt 膨胀风险。
- 文件：`txtToWorldbook/services/worldbookService.js`
   - 增量合并内容由“前50字包含”升级为近重复检测 + 段落去重合并。

#### 5) 角色四级分类贯通
- 文件：`txtToWorldbook/core/constants.js`
   - 角色内容指引与系统提示词增加 `角色类型` 强制要求。
- 文件：`txtToWorldbook/services/parserService.js`
   - 解析后统一归一 `角色类型`，缺失时默认 `普通配角`。
- 文件：`txtToWorldbook/ui/renderer.js`
   - 世界书条目卡片展示角色类型徽标。
- 文件：`txtToWorldbook/services/exportFormatService.js`
   - 导出格式补充 `storyweaverRoleType` 元数据。
- 文件：`txtToWorldbook/services/importExportService.js`
   - 角色卡导出 `extensions.character_role_type`。

### 验证结论
- 已完成静态错误检查：本次修改文件均无语法/类型报错。
- 行为层回归（手测）建议：
   1. 验证 `_卷N` 类重复条目在“别名合并”前置阶段自动消解；
   2. 验证“整理条目”对重复字段（如姓名/性别）不再重复堆叠；
   3. 验证长章节串行处理时 prompt 体积不再随世界书线性膨胀；
   4. 验证角色卡导出后可读取 `character_role_type`.
